bfs

  • bfs<T>(root: T, children: ChildrenGetter<T>, onNode: (node: T, context: BfsContext, path: T[], index: number) => boolean | void, onLevelStart?: (depth: number) => boolean | void): void

  • Breadth first traverse a tree like object.

    example
    type Node = {
      name: string
      checked?: boolean
      nodes?: Node[]
    }

    const tree: Node = {
      name: "A",
      nodes: [
        {
          name: "B",
          nodes: [
            { name: "C" },
            {
              name: "D",
              checked: true,
              nodes: [{ name: "E", checked: true }, { name: "Z" }],
            },
          ],
        },
        {
          name: "F",
          nodes: [{ name: "G" }, { name: "H", checked: true }],
        },
      ],
    }

    const nodes = []
    bfs(
      tree,
      (node: any) => node.nodes,
      (node, context) => {
        if (node.checked) {
          nodes.push(node.name)
          context.cut = true
        }
      },
    )
    nodes // ["D", "H"]

      Handling of the node. Return true if you want to interrupt the whole traversal, or use context.cut to indicate you don't want to go down this path.

dfs

  • dfs<T, Param, Return>(root: T, children: ChildrenGetter<T>, param: Param, __namedParameters?: DfsOptions<T, Param, Return>): undefined | Return

  • This is a general and efficient iterative version of depth first search/traversal.

    example
    type ExprNode = {
      op?: "+" | "-" | "*"
      val?: number
      children?: ExprNode[]
    }

    const expr: ExprNode = {
      op: "+",
      children: [
        {
          op: "*",
          children: [{ val: 3 }, { val: 4 }, { val: 1 }],
        },
        {
          op: "-",
          children: [{ val: 7 }, { val: 3 }],
        },
      ],
    }

    const ops = {
      "+": (nums: number[]) => nums.reduce((result, x) => result + x),
      "-": (nums: number[]) => nums.reduce((result, x) => result - x),
      "*": (nums: number[]) => nums.reduce((result, x) => result * x),
    }

    const ret = dfs<ExprNode, null, number>(
      expr,
      (node) => node.children,
      null,
      {
        onLeaf(node, context) {
          context.returnValue = node.val
        },
        onPost(node, context) {
          context.returnValue = ops[node.op!](context.childReturns)
        },
      },
    )

    Type parameters

    • T

      Node type

    • Param

      Parameters type

    • Return

      The return value type

    Parameters

    • root: T

      The root node of a tree like object to begin search with.

    • children: ChildrenGetter<T>

      An accessor function that returns the sub-nodes if any.

    • param: Param

      Optional parameters that can be used during traversal. Give it a null if you don't need it.

    • __namedParameters: DfsOptions<T, Param, Return> = {}

    Returns undefined | Return

    The evaluation result of the traversal.

rec2iter

  • rec2iter<Param, Return>(start: (param: Param, context: StartContext<Param>) => void | Return, consume: (param: Param, returns?: Return[], locals?: Locals) => Return, param: Param): Return

  • It implements an iterative execution of the given recursion. That is, you write an algorithm in a recursive way, but you get an iterative execution of it.

    The running is divided into 2 phases. In start phase, you write everything that runs before recursions, you also specify recursions you want to run here; In consume phase, you consume all the recursive calls' returns.

    NOTE: Because it has only 2 phases, it can't support all kinds of recursion usage, for more complex usages you can use recurse. For simpler ones however, this function is faster.

    example
    function factorial(n: number) {
      return rec2iter(
        (n, context) => {
          if (n <= 1) return 1
          context.recursions = [n - 1]
        },
        (n, [fac_n_minus_1]) => n * fac_n_minus_1,
        n,
      )
    }

    Type parameters

    • Param

    • Return

    Parameters

    • start: (param: Param, context: StartContext<Param>) => void | Return

      The start phase where you write everything that runs before recursive calls. You also specify what recursions you want to run next here.

        • (param: Param, context: StartContext<Param>): void | Return

        • Parameters

          • param: Param
          • context: StartContext<Param>

          Returns void | Return

    • consume: (param: Param, returns?: Return[], locals?: Locals) => Return

      The consume phase where you can get all recursive calls' return values.

        • (param: Param, returns?: Return[], locals?: Locals): Return

        • Parameters

          • param: Param
          • Optional returns: Return[]
          • Optional locals: Locals

          Returns Return

    • param: Param

      The initial parameters to kick off the recursion.

    Returns Return

    The result of the whole recursion.

recurse

  • recurse<Param, Return>(body: (param: Param) => Generator<Param, Return, Return>, initialParam: Param): Return

  • It implements an iterative execution of the given recursion. That is, you write an algorithm in a recursive way, but you get an iterative execution of it.

    example
    function factorial(n: number) {
      return recurse<number, number>(function* (n) {
        if (n <= 1) return 1
        return n * (yield n - 1)
      }, n)
    }
